## Overall Impression

Structure is correct: summary first, nav, then cases. Every design decision below that structure was left at zero. Dense, colorless, exhausting to read for more than two cases. Single biggest opportunity: make the verdict and DET/LLM agreement state immediately readable within each case without scanning.

## What's Working

1. **Summary table at the top.** Front-loading the 20-case overview is correct. The QI reviewer can orient before diving in.
2. **Side-by-side DET/LLM verdict layout.** Two-column `.vbox det` / `.vbox llm` structure is the right abstraction for direct comparison.
3. **Progressive disclosure via `<details>`.** Reasoning sections collapsed by default is the right call for this density.

## Priority Issues

**[P1] Two banned side-stripe borders**
- `.meta { border-left: 4px solid #888 }` and `.note { border-left: 3px solid #aaa }`
- Fix: `.meta` — `border: 1px solid oklch(90% 0.012 28); background: oklch(97% 0.008 28)`. `.note` — `border: 1px solid oklch(87% 0.015 70); background: oklch(96% 0.04 70)`.

**[P1] Summary table: 10 equal-weight columns, no verdict emphasis**
- HN/AN columns get same weight as Deterministic/LLM; no visual marker for discrepant cases.
- Fix: De-emphasize HN/AN to 11px muted. Bold or tint the verdict columns. Add row background when DET≠LLM.

**[P1] No visual hierarchy within case sections**
- Verdict pill (18px bold) surrounded by equally-weighted prose. H2 background-fill adds noise.
- Fix: Elevate the verdict area; make the case H2 a clean typographic break only; verdict pill should be visual apex.

**[P2] Nursing notes at 12px full-width are unreadable**
- 12px `<pre>` across 1200px; Thai content has no elevated line-height.
- Fix: `pre { font-size: 13px; line-height: 1.6; max-width: 80ch; }`. Thai blocks: `line-height: 1.7`.

**[P2] No classification legend and unexplained system labels**
- `(deterministic-final)` appears 12 times unexplained. 5 classification states undefined. Confidence score scale missing.
- Fix: Add compact legend below summary table (or collapsible `<details>`).

## Persona Red Flags

**Alex (Power User / KCMH Developer):** Wants to find all DET≠LLM cases in 30 seconds. No such path exists. No collapse-all means 60+ clicks to read all reasoning. Will abandon after case 5.

**Clinical Reviewer (QI Committee Member):** Doesn't know what `hb_7_to_10` or `(deterministic-final)` means. No statement of what to do with this page. No signal that it's read-only (no action path, but also no explanation).

**Sam (Accessibility-Dependent User):** Summary table has no `<caption>`, no `scope` on headers. Classification pills convey meaning through color alone (no `aria-label`). Screen reader gets column headers but no pill semantics.

## Minor Observations

- `border-bottom: 2px solid #333` on `h1` lands cleanly; keep it.
- `.vbox.det` (#f8f8ff) / `.vbox.llm` (#fff8f0) color distinction is functionally correct; worth migrating to OKLCH.
- Confidence score `conf` span renders at 12px next to 18px bold — hierarchy inverted. Model name should be de-emphasized; confidence should be promoted.
- H2 summary inline style override is a symptom of an incorrect global rule. Normalize the H2.
