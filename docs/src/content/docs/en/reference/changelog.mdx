## Status

30 modules under `bba.*` are merged to `main`, `version = "0.1.0"` in
`pyproject.toml`. The `v0.1.0` tag itself is pending — the maintainer will
trigger it explicitly with shared-state intent.

Until the tag fires, this page is a stub. The authoritative changelog
will live at
[GitHub Releases](https://github.com/macsermkiat/blood_audit/releases).

## What "Phase 1 (RBC) feature-complete" includes

- 12-stage audit pipeline modules (ingest → audit_store); CLI wiring is staged.
- Deterministic 3-tier Hb classifier with cohort-aware thresholds.
- Acute-blood-loss override with documented delta-Hb gates.
- LLM_REVIEW pathway with Anthropic Batch API + Sonnet→Opus escalation.
- Six fail-closed quote-grounding layers.
- Audit store with run-level idempotency.
- Eval harness primitives.
- Monitoring (sentinel κ-statistic, golden-set drift SPRT).
- Dashboard, report generator, CLI scaffold.
- CI: ruff + mypy strict (src) + pytest with testcontainers Postgres.

## What shipped after the initial Phase 1 build

- **Phase 2 — platelet auditor (PR #85, merged).** `component_map` routes each
  order to a component family (RED_CELL / PLATELET / FFP / CRYO / WHOLE_BLOOD /
  UNKNOWN); `platelet_lookup` + `platelet_classifier` add the §5.1 count gate.
  The gate auto-clears nothing: count ≥ 100 ×10³/µL → review, count < 100 →
  defer to LLM, missing → INSUFFICIENT_EVIDENCE. The platelet LLM leg is behind
  the default-off `PLATELET_LLM_ENABLED` flag; the RBC path is byte-identical.
- **Doctor / department attribution (Feature 2, PR #82).** `bba.attribution`
  ranks the top-10 ordering doctors and departments by appropriateness in three
  buckets, via `BDVST.DCTREQ` → `DCT.csv` with a swappable verdict source.
- **Cohort refinements.** `ORTHO_SURGERY` split into its own cohort (Hb floor
  8.0, PR #83); `CARDIOPULMONARY_COMORBIDITY` cohort + peri-op defer-to-LLM
  (PR #81).
- **Stale-volume gate on the bleeding auto-clear.** The RBC over-clear
  guardrail's only prose exemption (a grounded `ACTIVE_BLEEDING` quote at
  confidence ≥ 0.8 with a documented volume > 300 mL or a life-threatening
  marker) now blanks quote spans charted under a `d/m/y` date anchor (Thai BE
  or CE) more than 7 days before the order, so a weeks-old quantified index
  bleed can no longer clear the current order. Undated or unparseable text is
  untouched — the gate can only withhold the exemption. The RBC prompt states
  the same current-episode rule so the model's verdict agrees with the
  guardrail.
- **Melena disqualified from the volume exemption.** Melena is digested /
  old blood, not active bleeding: it never satisfies the > 300 mL
  active-bleeding volume exemption; such orders clear only via shock or
  low-Hb evidence.
- **Returns ledger (spec #119; go-live PR #138).** `bba.returns_ledger`
  aggregates the per-unit `BDVSTTRANS` disposition export. Orders whose
  units were all returned un-transfused terminal as
  `RETURNED_NOT_TRANSFUSED`; peri-op transfused orders as
  `PERIOP_TRANSFUSION_EXEMPT`. Both are excluded from appropriateness
  scoring. Default-ON (`RETURNS_LEDGER_ENABLED`); the platelet leg shares
  the same short-circuit. `BDVSTTRANS` became the 13th canonical ingest
  table.
- **Administration-confirmation gate (spec #105).** Reserve-ahead orders
  whose transfusion cannot be confirmed terminal as
  `PREOP_RESERVATION_UNCONFIRMED` instead of receiving an Hb verdict —
  behind the default-off `RESERVE_AHEAD_ROUTER_ENABLED` flag pending the
  clinical gate (#109).
- **Declared-use USETYPE routing (spec #147; PRs #148–#152, #194, #195).**
  `bba.declared_use` interprets order-time `BDVSTDT.USETYPE`. Orders
  whose `USETYPE` code declares surgical use (`2` → `surgery`) or
  type-and-screen (`3` → `type_screen`) are exempt at the
  order level (`preop_declared_exempt` → `PERIOP_TRANSFUSION_EXEMPT`), and
  USETYPE is now the **only** pre-op router — the legacy bare-surgical
  ≤ 6 h / ≤ 72 h deterministic arms are retired while the default-ON flags
  (`DECLARED_USETYPE_ENABLED`, `DECLARED_USE_PREOP_EXEMPT_ENABLED`) hold.
- **MSBOS pre-op reservation audit (specs #159–#175).**
  `bba.preop_reservation` audits reserved units for planned operations
  against the MSBOS / OPBloodLimit reference; over-reservation surfaces as
  `PREOP_OVER_RESERVATION`. The platelet leg screens against
  clinician-signed count cutoffs (MNS 80k, neuraxial 100k, cardiac-CPB
  100k). Never-guess rules route unresolved / ambiguous / missing-count
  cases to review. Default-ON (`MSBOS_RESERVATION_ENABLED`).
- **Returns-terminal MSBOS annotation (spec #176; PRs #181–#184).**
  Returned / peri-op-exempt orders that carried a pre-op reservation now
  embed a `reservation_annotation` (annotation, never a verdict change) so
  crossmatch-to-transfusion waste stays visible on excluded rows.
- **MSBOS name-match offline study (spec #185; PRs #190–#193).** Tier-1
  deterministic + Tier-2 LLM procedure-name matching for reservation cases
  the coded pickers cannot resolve. Annotation-only offline study; not
  pipeline-wired.
- **Planned-op picker v2 + OPRTACT→ICD9 bridge (spec #196; PRs #202–#216).**
  The MSBOS arm now picks the planned operation inside a 72-hour window via
  a vendored OPRTACT→ICD9 bridge reference, with a guarded-prefix denylist,
  a dominance-ceiling rule, and picker provenance embedded on every
  reservation marker; bridge disagreements gate the verdict to review.
  Default-ON (`MSBOS_PLANNED_OP_PICKER_V2_ENABLED`).
- **CLI `bba report` wired.** Renders committee CSVs, per-physician CSVs,
  and the PDF from the audit store; requires `BBA_BDVST_CSV` /
  `BBA_DCT_CSV` for the attribution resolvers. The report aggregate
  models gained an `over_reservation_count` field (not yet serialized as
  a CSV column). `evaluate`, `serve-dashboard`, and `sentinel` remain
  integration seams.
- **Refactors (#218, #222, #223).** Single home for the MSBOS reservation
  reason vocabulary (`preop_reservation.reason_presentation`); a typed
  derived seam for reserve-ahead / declared-exempt routing; and
  `_build_audit_row` guardrails folded into an ordered list.
