## Status

26 modules under `bba.*` are merged to `main`, `version = "0.1.0"` in
`pyproject.toml`. The `v0.1.0` tag itself is pending — the maintainer will
trigger it explicitly with shared-state intent.

Until the tag fires, this page is a stub. The authoritative changelog
will live at
[GitHub Releases](https://github.com/macsermkiat/blood_audit/releases).

## What "Phase 1 (RBC) feature-complete" includes

- 12-stage audit pipeline modules (ingest → audit_store); CLI wiring is staged.
- Deterministic 3-tier Hb classifier with cohort-aware thresholds.
- Acute-blood-loss override with documented delta-Hb gates.
- LLM_REVIEW pathway with Anthropic Batch API + Sonnet→Opus escalation.
- Six fail-closed quote-grounding layers.
- Audit store with run-level idempotency.
- Eval harness primitives.
- Monitoring (sentinel κ-statistic, golden-set drift SPRT).
- Dashboard, report generator, CLI scaffold.
- CI: ruff + mypy strict (src) + pytest with testcontainers Postgres.

## What shipped after the initial Phase 1 build

- **Phase 2 — platelet auditor (PR #85, merged).** `component_map` routes each
  order to a component family (RED_CELL / PLATELET / FFP / CRYO / WHOLE_BLOOD /
  UNKNOWN); `platelet_lookup` + `platelet_classifier` add the §5.1 count gate.
  The gate auto-clears nothing: count ≥ 100 ×10³/µL → review, count < 100 →
  defer to LLM, missing → INSUFFICIENT_EVIDENCE. The platelet LLM leg is behind
  the default-off `PLATELET_LLM_ENABLED` flag; the RBC path is byte-identical.
- **Doctor / department attribution (Feature 2, PR #82).** `bba.attribution`
  ranks the top-10 ordering doctors and departments by appropriateness in three
  buckets, via `BDVST.DCTREQ` → `DCT.csv` with a swappable verdict source.
- **Cohort refinements.** `ORTHO_SURGERY` split into its own cohort (Hb floor
  8.0, PR #83); `CARDIOPULMONARY_COMORBIDITY` cohort + peri-op defer-to-LLM
  (PR #81).
