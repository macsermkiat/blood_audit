## Bypass and review paths

The plain Hb tier is not the first check. Two order-level routing exits
run before any Hb reasoning (both default-ON):

- **All units returned un-transfused** (per the BDVSTTRANS unit ledger)
  → `RETURNED_NOT_TRANSFUSED`. No transfusion occurred; the order is
  excluded from appropriateness scoring (it remains visible as
  crossmatch-to-transfusion waste).
- **Declared surgical use / type-and-screen** (`BDVSTDT.USETYPE` `M/G` or
  `T/S`) → `PERIOP_TRANSFUSION_EXEMPT`. The clinician's declared use is
  authoritative; peri-op transfusions are not judged against the ward Hb
  policy. Also excluded from scoring.

After excluding missing Hb, the algorithm applies the global low-Hb rule
and then looks for other high-precedence evidence:

- **Hb < 7.0 g/dL** → deterministic `APPROPRIATE` before cohort routing.
- **MTP clinical context** → deterministic `APPROPRIATE`.
- **Delta-Hb trigger** → deterministic `APPROPRIATE` when the Hb drop meets:
  - ≥ 1.5 g/dL drop within 6 hours
  - ≥ 2.0 g/dL drop within 12 hours
  - ≥ 2.5 g/dL drop within 24 hours
- **Hemodilution or isolated low Hb without trend** → `NEEDS_REVIEW` only
  after the Hb < 7.0 rule has already had first chance to classify.
- **Unknown or non-threshold clinical context** → `NEEDS_REVIEW` when
  Hb is ≥ 7.0.

Legacy mode only (when the declared-use flag is forced off): a
peri-procedural anchor ≤ 6h before the order auto-clears, and a pre-op
crossmatch ≤ 72h before an upcoming operation **defers to the LLM**
rather than auto-clearing — a reservation is not an indication.

Vital-sign abnormalities alone (low BP, tachycardia) do not trigger the
bypass.

## Pre-op reservation screening (MSBOS)

Separately from the transfusion verdict, orders reserved for a planned
operation are screened against the MSBOS / OPBloodLimit reference
(default-ON). Reserving more units than the recommendation for the
planned procedure surfaces as `PREOP_OVER_RESERVATION`; the platelet leg
compares the pre-op platelet count against clinician-signed cutoffs
(MNS 80, neuraxial 100, cardiac-CPB 100, ×10³/µL). The screen never
guesses: unresolved procedures, ambiguous codes, and missing counts route
to review instead of auto-flagging.

## Exclusions and insufficient evidence

- Pediatric patients (age < 15) — different policy applies.
- Non-RBC products (LPRC, LDPRC, SDR).
- ICD-10 conditions that take the patient out of scope per PR 17.2.

Orders without a documented pre-transfusion Hb are **not** classified as
appropriate or inappropriate. They are retained as `INSUFFICIENT_EVIDENCE`
so the QI committee can see the documentation gap.
