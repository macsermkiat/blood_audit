## The 12-stage happy path

<PipelineDiagram />

Each stage is an importable subpackage under `src/bba/`. The `bba.audit_pipeline`
module orchestrates them; no other module reaches across stage boundaries.

Intake forks at `bba.component_map`: red-cell orders take the 12-stage RBC path
below (Phase 1), and platelet orders take the platelet leg (Phase 2 â€” merged,
PR #85). The RBC path is byte-identical whether or not the platelet feature is
enabled.

## Invariants worth keeping

1. **Deterministic-first.** The deterministic classifier is authoritative.
   The LLM route is entered only after the deterministic classifier returns
   `NEEDS_REVIEW` or `POTENTIALLY_INAPPROPRIATE`.
2. **Idempotent re-runs.** `run_id = sha256(input_csv_hash +
   schema_fingerprint + code_version)`. `bba audit` is a no-op on a
   completed `run_id` unless `--force` is passed (which writes an
   `idempotency_override=true` row to `audit_log`).
3. **Quote-grounded LLM output.** All six grounding layers run; failure
   sets `final_classification = NEEDS_REVIEW` with `review_reason =
   "hallucination_suspect"`, never an `APPROPRIATE` / `INAPPROPRIATE`
   result row.
4. **No real-time path.** No subscriber to the audit stream emits
   clinician alerts. This is QI, not SaMD.
5. **De-identification happens before any cross-container hop.** The
   `bba.deid_redactor` step is non-optional.

## Where the integration seams live

`bba.dashboard`, `bba.eval_harness`, and the sentinel orchestration in
`bba.monitoring` ship complete but are not wired to the CLI by default
(`bba report` is now wired to `bba.report_generator`). See
[Integration seams](/blood_audit/en/operators/integration-seams/).
