When the deterministic classifier routes a row to LLM review, the LLM
emits a clinical reasoning summary along with quoted evidence from the
de-identified bundle. Every quote passes through six mandatory
fail-closed layers; a seventh medical-NLI semantic-entailment layer is
opt-in.

## What "fail-closed" means

Any of the six mandatory layers that returns false marks the row as
`final_classification = NEEDS_REVIEW` with `review_reason =
"hallucination_suspect"`. The row is never silently downgraded, retried
with relaxed constraints, or accepted as an `APPROPRIATE` /
`INAPPROPRIATE` result. The QI committee can filter on
`review_reason = "hallucination_suspect"` to audit grounding failures
separately.

`hallucination_suspect` is a `review_reason` flag, not a classification
literal. The stored `Classification` literals are `APPROPRIATE`,
`INAPPROPRIATE`, `NEEDS_REVIEW`, `INSUFFICIENT_EVIDENCE`,
`POTENTIALLY_INAPPROPRIATE` — see [Audit store](/blood_audit/en/developers/audit-store/).

## The medical-NLI layer is opt-in

`bba.quote_grounder.verify_citation(..., nli_gate=None)` defaults to
**no NLI check**. The verifier only evaluates the medical-NLI layer
when the caller passes a non-null `NLIEntailmentGate`. Default call
sites in the repo do not pass one, so the layer is not part of the
mandatory fail-closed stack today.

PRD §9 specifies medical-NLI as opt-in. Deployments that want the
extra semantic check construct an `NLIEntailmentGate` (typically over a
hosted ClinicalBERT-NLI checkpoint) and pass it into the verifier.

## What you may not do

- Mock or short-circuit any layer in tests.
- Add an opt-out flag to skip grounding for "fast iteration".
- Lower the 25-character minimum.
- Replace the medical-NLI step with a bare string check.

If you find yourself wanting to do any of the above, the right answer is
to write a smaller test fixture, not to weaken the contract.
