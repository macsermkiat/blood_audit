When the deterministic classifier routes a row to LLM review, the LLM
emits a clinical reasoning summary along with quoted evidence from the
de-identified bundle. **Every quote** is gated by the layers below before
the row is accepted into the audit store.

<QuoteGroundingLayers />

## What "fail-closed" means

Any layer that returns false marks the row as `final_classification =
NEEDS_REVIEW` with `review_reason = "hallucination_suspect"`. The row is
never silently downgraded, retried with relaxed constraints, or accepted
as an `APPROPRIATE` / `INAPPROPRIATE` result. The QI committee can
filter on `review_reason = "hallucination_suspect"` to audit grounding
failures separately.

`hallucination_suspect` is a `review_reason` flag, not a classification
literal. The stored `Classification` literals are `APPROPRIATE`,
`INAPPROPRIATE`, `NEEDS_REVIEW`, `INSUFFICIENT_EVIDENCE`,
`POTENTIALLY_INAPPROPRIATE` — see [Audit store](/blood_audit/en/developers/audit-store/).

## What you may not do

- Mock or short-circuit any layer in tests.
- Add an opt-out flag to skip grounding for "fast iteration".
- Lower the 25-character minimum.
- Replace the medical-NLI step with a bare string check.

If you find yourself wanting to do any of the above, the right answer is
to write a smaller test fixture, not to weaken the contract.
