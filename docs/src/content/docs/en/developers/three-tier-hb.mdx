The deterministic classifier is the **authoritative** decision layer for
auto-classified rows. `APPROPRIATE` and `INSUFFICIENT_EVIDENCE` are final
without an LLM call. `NEEDS_REVIEW` and `POTENTIALLY_INAPPROPRIATE` route
to the LLM for positive-evidence reasoning and quote grounding.

## What "cohort" means here

In this project, a **cohort** is the clinical-context bucket assigned to
one transfusion order before the Hb number is interpreted.

It is not a research cohort and it is not a permanent patient label. It is
a deterministic rule output for one audit row: "given the procedures,
diagnoses, medications, blood-order pattern, and available data around this
order, which Hb threshold should the audit use?"

The same patient could theoretically land in different cohorts on different
admissions or different transfusion dates if the surrounding evidence changes.
For example, a patient with no current surgery signal might be `DEFAULT` on
one admission, but `CARDIAC_SURGERY` after a later cardiac operation.

The cohort matters because the audit does **not** use one Hb threshold for
everyone:

- Some orders use a numeric Hb threshold, such as 7.0, 7.5, or 8.0 g/dL.
- Some orders do not have a numeric threshold because the context is special,
  such as MTP or active heme malignancy.
- Some orders are routed to review because the required data is missing or
  the context should not be reduced to one number.

## Cohort examples in plain language

| Cohort | What it means | What data triggers it | What the classifier does |
| --- | --- | --- | --- |
| `DEFAULT` | No special cohort rule matched. | `CARDIAC_SURGERY` | The order is near a recent cardiac / large-vessel operative context. | Recent operating-room procedure with cardiac / major vascular ICD-9-CM procedure-code prefixes, with non-OR cardiac items excluded. | Uses Hb threshold **7.5 g/dL**. |
| `ORTHO_SURGERY` | Orthopedic surgery on its own. A higher threshold than default. | Orthopedic operative procedure (ICD-9-CM prefixes **78**, **79**, **81**) within 30 days before the order. Cardiac history is **not** required. | Uses Hb threshold **8.0 g/dL**. Ranked above `CARDIAC_SURGERY`, so when both surgical contexts are present the higher floor wins. |
| `ESRD_EPO` | End-stage renal disease context with dialysis-medication evidence. | ESRD ICD-10 diagnosis plus a dialysis-context medication signal. Both are required. | Uses Hb threshold **8.0 g/dL**. |
| `CARDIOPULMONARY_COMORBIDITY` | Chronic **heart-disease** comorbidity, judged from diagnoses rather than a surgery. | `MTP` | Massive transfusion protocol pattern. This is not judged by the Hb threshold. | RBC-unit cluster pattern, or RBC + FFP + platelet co-ordering pattern in the MTP window. | Bypasses the Hb tier and returns `APPROPRIATE`. |
| `HEME_MALIGNANCY_ACTIVE` | Active hematologic malignancy context where one numeric threshold is not enough. | Heme-malignancy ICD-10 signal plus recent chemotherapy medication plus neutropenia. | Hb < 7.0 is `APPROPRIATE`; otherwise no numeric threshold is applied and the row routes to `NEEDS_REVIEW` / LLM context review. |
| `UNKNOWN` | The audit does not have enough procedure data to safely choose a threshold. | Procedure events are unavailable, not merely empty. | Hb < 7.0 is `APPROPRIATE`; otherwise the classifier does not silently assume 7.0 and routes to `NEEDS_REVIEW`. |

> The older `ORTHO_CARDIAC` cohort (orthopedic surgery **plus** cardiac
> history) is **deprecated**: an orthopedic operation now raises the floor to
> 8.0 on its own, which subsumes the fused cohort. Its threshold is retained
> only so audit-store rows written before the split still resolve.

The key safety idea: **missing procedure data is different from no procedure
found**. If procedure data is available and empty, the detector can fall
through to `DEFAULT`. If procedure data is unavailable, the detector returns
`UNKNOWN` so a human can review the order.

## Exact data checked by step

The classifier runs in a fixed order. The first matching rule wins. This is
the operational checklist the audit follows for one RBC order:

| Step | Question the code asks | Source data checked | Important details | Output if yes |
| --- | --- | --- | --- | --- |
| 1 | Is there no usable pre-transfusion Hb? | **Lab** table Hb observations. | The Hb lookup searches Hb results for the same patient at or before the order time. It only accepts analytic-valid Hb values in **2-25 g/dL** and looks back **less than 7 days** (`anchor - Hb time < 7 days`). Hematology Hb is preferred over POCT when available. | 2 | Is Hb < 7.0 g/dL? | **Lab** current Hb. | This global low-Hb rule runs before cohort-specific review routes, including `UNKNOWN`, active heme malignancy, hemodilution, and single-low-Hb-no-trend. | `APPROPRIATE`, final without LLM. |
| 3 | Is procedure data missing? | Procedure-event join availability. | This is not the same as "no procedure found." If the procedure join is unavailable and Hb is â‰Ą 7.0, the audit cannot safely know whether cardiac/orthopedic procedure rules apply. | `NEEDS_REVIEW`, routed to LLM/human review. |
| 4 | Does the order look like massive transfusion protocol (MTP)? | Blood-order history around the anchor: RBC units, FFP co-order flag, platelet co-order flag. | After procedure-event availability is confirmed, the detector checks the **1-hour window before the order**. | 5 | Is there a procedure within 6 hours before the order? | **IPTSUMOPRT** procedure rows joined to **ICD9CM** procedure dictionary. | The pipeline computes `procedure_proximity_hours` from operative procedure timing. | 7 | Did Hb drop fast enough to suggest acute blood loss? | **Lab** prior Hb observations. | The current Hb is compared with the highest prior Hb in each window. Any one trigger is enough: drop **â‰Ą 1.5 g/dL in 6h**, **â‰Ą 2.0 in 12h**, or **â‰Ą 2.5 in 24h**. | `APPROPRIATE`, final without LLM. |

| 10 | Is Hb in the gray zone? | **Lab** current Hb plus cohort threshold. | Gray zone means **cohort threshold â‰¤ Hb < 10 g/dL**. The deterministic layer does not make the final clinical call here. | `NEEDS_REVIEW`, routed to LLM/human review. |
| 11 | Is Hb high, â‰Ą 10 g/dL? | **Lab** current Hb. | High pre-transfusion Hb is flagged as concerning, but the LLM can look for documented positive evidence before final publication. | `POTENTIALLY_INAPPROPRIATE`, routed to LLM/quote grounding. |

## Cohort assignment checklist

The cohort detector runs before the Hb tier comparison. It uses a separate
fixed order:

| Cohort check order | Cohort if matched | Data checked | Exact rule |
| --- | --- | --- | --- |
| 1 | `UNKNOWN` | Procedure-event join | If procedure events are unavailable (`None`), stop. Do not apply the default 7.0 threshold. |
| 2 | `MTP` | Blood-order events | In the prior 1 hour: total RBC units â‰Ą 4, or RBC + FFP + platelets co-ordered. |
| 3 | `ORTHO_SURGERY` | **ICD-9-CM procedure** | Orthopedic OR procedure within 30 days before the order, using ICD-9-CM procedure prefixes **78**, **79**, or **81**. Cardiac history is **not** required. Checked before `CARDIAC_SURGERY` so the higher floor wins when both are present. Threshold 8.0. |
| 4 | `CARDIAC_SURGERY` | **ICD-9-CM procedure** | Cardiac / large-vessel OR procedure within 30 days before the order. ICD-9-CM procedure prefixes **36**, **38**, or **39**. Non-OR cardiac items **894** and **3796** are excluded. Threshold 7.5. |
| 5 | `ESRD_EPO` | **ICD-10 diagnosis** + **MED** drug text | ESRD diagnosis **N18.5** or **N18.6** plus recent dialysis-context medication within 14 days. Medication keywords include `heparin`, `sevelamer`, `cinacalcet`. Threshold 8.0. |
| 6 | `HEME_MALIGNANCY_ACTIVE` | **ICD-10 diagnosis** + **MED** drug text + **Lab ANC** | Heme malignancy ICD-10 prefix **C8** or **C9**, plus recent chemo medication within 30 days, plus ANC < 500 cells/uL. No numeric threshold; route to review. |
| 7 | `CARDIOPULMONARY_COMORBIDITY` | **ICD-10 diagnosis** | Heart-disease diagnosis prefix **I11**, **I13**, **I20-I25**, **I42**, or **I50** (diagnosis-driven, no surgery required). Lung-disease diagnoses are excluded. Threshold 8.0. |
| 8 | `DEFAULT` | Fall-through after all above checks | Procedure data is available, but no special cohort matched. Threshold 7.0. |

## Source-to-rule map

- **Lab**: current Hb, prior Hb trend, delta-Hb windows, Hb freshness, ANC for
  active heme-malignancy context.
- **ICD-9-CM procedure codes**: cardiac surgery, orthopedic surgery,
  peri-procedural timing, pre-op crossmatch timing. Procedure rows come from
  `IPTSUMOPRT`; OR/non-OR status and procedure names come from `ICD9CM`.
- **ICD-10 diagnosis codes**: cardiac history, ESRD, heme malignancy, and
  heart-disease comorbidity (the `CARDIOPULMONARY_COMORBIDITY` 8.0 g/dL floor).
- **MED**: dialysis-medication signals, chemotherapy-medication signals,
  and crystalloid volume in the 4 hours before the order.
- **Blood-order history**: MTP detection from RBC-unit clustering or
  RBC + FFP + platelet co-ordering.

## Precedence order (top wins)

The cohort-specific Hb-tier rule is late in the classifier. The global
Hb < 7.0 rule is early, and later checks can still short-circuit the result:

1. `Hb missing` â†’ `INSUFFICIENT_EVIDENCE`
2. Hb < 7.0 g/dL â†’ `APPROPRIATE`
3. Cohort `UNKNOWN` â†’ `NEEDS_REVIEW`
4. Cohort `MTP` (massive transfusion protocol) â†’ `APPROPRIATE`
5. Peri-procedural anchor â‰¤ 6h â†’ `APPROPRIATE`
6. Pre-op crossmatch / upcoming procedure â‰¤ 72h â†’ `APPROPRIATE`
7. Delta-Hb trigger â†’ `APPROPRIATE`
8. Non-threshold cohort â†’ `NEEDS_REVIEW`
9. Hemodilution (Hb < threshold + â‰Ą 2L crystalloid / 4h) â†’ `NEEDS_REVIEW`
10. Single-low-Hb-no-trend (isolated Hb < 8 + no prior 24h Hb) â†’ `NEEDS_REVIEW`
11. Hb-tier rule (the matrix above)

## The bypass paths to `APPROPRIATE`

Four bypasses can return `APPROPRIATE` before the plain Hb-tier rule:

- **MTP cohort** â€” the massive-transfusion-protocol pattern is already
  clinically load-bearing enough to stop the deterministic classifier.
- **Peri-procedural â‰¤ 6h** â€” the order is within 6 hours after an active
  surgical or procedural episode.
- **Pre-op crossmatch â‰¤ 72h** â€” the order is within 72 hours before an
  upcoming operative event, consistent with surgical preparation.
- **Delta-Hb trigger** (the acute-blood-loss override) â€” a documented
  Hb drop meets one of:
  - â‰Ą 1.5 g/dL drop within 6 hours
  - â‰Ą 2.0 g/dL drop within 12 hours
  - â‰Ą 2.5 g/dL drop within 24 hours

## Review flags before auto-appropriate

After the global Hb < 7.0 rule, two checks can stop a would-be
sub-threshold `APPROPRIATE` row and send it to review instead:

- **Hemodilution flagged** â€” Hb is below threshold, but â‰Ą 2 L crystalloid
  in the prior 4 hours makes the Hb value suspect.
- **Single-low-Hb-no-trend** â€” an isolated Hb < 8 g/dL has no prior 24-hour
  Hb trend to confirm the anemia pattern.

Vital signs (low BP, high HR) are **supporting evidence only**. They
never trigger a bypass on their own and never override the Hb tier.

## Threshold summary

| Cohort | Hb threshold for `APPROPRIATE` | Notes |
| --- | --- | --- |
| Default | < 7.0 g/dL | Fall-through threshold. |
| Cardiac surgery | < 7.5 g/dL | Recent cardiac / large-vessel OR procedure. |
| Orthopedic surgery | < 8.0 g/dL | Orthopedic OR procedure on its own; cardiac history not required. |
| ESRD + dialysis-med signal | < 8.0 g/dL | ESRD diagnosis plus active dialysis-context medication. |
| Heart-disease comorbidity | < 8.0 g/dL | Diagnosis-driven (`CARDIOPULMONARY_COMORBIDITY`): ICD-10 heart-disease code, no surgery required. |
| MTP | None | Bypasses directly to `APPROPRIATE`. |
| Active heme malignancy | None | Hb < 7.0 is `APPROPRIATE`; Hb â‰Ą 7.0 routes to `NEEDS_REVIEW`; LLM interprets context. |
| Unknown | None | Hb < 7.0 is `APPROPRIATE`; Hb â‰Ą 7.0 routes to `NEEDS_REVIEW`; the classifier refuses to assume a threshold. |

> no longer emitted â€” `ORTHO_SURGERY` applies the same 8.0 floor to an
> orthopedic operation alone.

## Why vitals are not standalone

Hemoglobin is the published threshold in KCMH PR 17.2 and AABB 2023.
Vital-sign-driven transfusion is a discretionary clinical judgment
outside the audit's deterministic scope. Vitals and notes enter the LLM
evidence bundle or human-review surface when a row routes to review, but
only structured MTP, procedure-timing, pre-op crossmatch, or delta-Hb evidence
can auto-bypass the Hb tier.
