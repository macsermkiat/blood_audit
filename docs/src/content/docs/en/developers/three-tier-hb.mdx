The deterministic classifier is the **authoritative** decision layer for
auto-classified rows. `APPROPRIATE` and `INSUFFICIENT_EVIDENCE` are final
without an LLM call. `NEEDS_REVIEW` and `POTENTIALLY_INAPPROPRIATE` route
to the LLM for positive-evidence reasoning and quote grounding.

## What "cohort" means here

In this project, a **cohort** is the clinical-context bucket assigned to
one transfusion order before the Hb number is interpreted.

It is not a research cohort and it is not a permanent patient label. It is
a deterministic rule output for one audit row: "given the procedures,
diagnoses, medications, blood-order pattern, and available data around this
order, which Hb threshold should the audit use?"

The same patient could theoretically land in different cohorts on different
admissions or different transfusion dates if the surrounding evidence changes.
For example, a patient with no current surgery signal might be `DEFAULT` on
one admission, but `CARDIAC_SURGERY` after a later cardiac operation.

The cohort matters because the audit does **not** use one Hb threshold for
everyone:

- Some orders use a numeric Hb threshold, such as 7.0, 7.5, or 8.0 g/dL.
- Some orders do not have a numeric threshold because the context is special,
  such as MTP or active heme malignancy.
- Some orders are routed to review because the required data is missing or
  the context should not be reduced to one number.

## Cohort examples in plain language

| Cohort | What it means | What data triggers it | What the classifier does |
| --- | --- | --- | --- |
| `DEFAULT` | No special cohort rule matched. This is the ordinary restrictive-transfusion pathway. | Procedure data is available, but no higher-priority cohort evidence matched. | Uses Hb threshold **7.0 g/dL**. Hb below 7.0 can be `APPROPRIATE`; Hb 7.0-10 routes to review; Hb ≥10 is high-Hb concern. |
| `CARDIAC_SURGERY` | The order is near a recent cardiac / large-vessel operative context. | Recent operating-room procedure with cardiac / major vascular ICD-9-CM procedure-code prefixes, with non-OR cardiac items excluded. | Uses Hb threshold **7.5 g/dL**. |
| `ORTHO_CARDIAC` | Orthopedic surgery plus cardiac history. This has a higher threshold than default. | Orthopedic operative procedure plus cardiac-history ICD-10 evidence such as ischemic heart disease or heart failure. | Uses Hb threshold **8.0 g/dL**. |
| `ESRD_EPO` | End-stage renal disease context with dialysis-medication evidence. | ESRD ICD-10 diagnosis plus a dialysis-context medication signal. Both are required. | Uses Hb threshold **8.0 g/dL**. |
| `MTP` | Massive transfusion protocol pattern. This is not judged by the Hb threshold. | RBC-unit cluster pattern, or RBC + FFP + platelet co-ordering pattern in the MTP window. | Bypasses the Hb tier and returns `APPROPRIATE`. |
| `HEME_MALIGNANCY_ACTIVE` | Active hematologic malignancy context where one numeric threshold is not enough. | Heme-malignancy ICD-10 signal plus recent chemotherapy medication plus neutropenia. | Does **not** use a numeric threshold. Routes to `NEEDS_REVIEW` / LLM context review. |
| `UNKNOWN` | The audit does not have enough procedure data to safely choose a threshold. | Procedure events are unavailable, not merely empty. | Does **not** silently assume 7.0. Routes to `NEEDS_REVIEW`. |

The key safety idea: **missing procedure data is different from no procedure
found**. If procedure data is available and empty, the detector can fall
through to `DEFAULT`. If procedure data is unavailable, the detector returns
`UNKNOWN` so a human can review the order.

## Precedence order (top wins)

The Hb-tier rule is late in the classifier. Earlier checks can
short-circuit the result:

1. `Hb missing` → `INSUFFICIENT_EVIDENCE`
2. Cohort `MTP` (massive transfusion protocol) → `APPROPRIATE`
3. Cohort `UNKNOWN` → `NEEDS_REVIEW`
4. Peri-procedural anchor ≤ 6h → `APPROPRIATE`
5. Pre-op crossmatch / upcoming procedure ≤ 72h → `APPROPRIATE`
6. Delta-Hb trigger → `APPROPRIATE`
7. Non-threshold cohort → `NEEDS_REVIEW`
8. Hemodilution (Hb < threshold + ≥ 2L crystalloid / 4h) → `NEEDS_REVIEW`
9. Single-low-Hb-no-trend (isolated Hb < 8 + no prior 24h Hb) → `NEEDS_REVIEW`
10. Hb-tier rule (the matrix above)

## The bypass paths to `APPROPRIATE`

Four bypasses can return `APPROPRIATE` before the plain Hb-tier rule:

- **MTP cohort** — the massive-transfusion-protocol pattern is already
  clinically load-bearing enough to stop the deterministic classifier.
- **Peri-procedural ≤ 6h** — the order is within 6 hours after an active
  surgical or procedural episode.
- **Pre-op crossmatch ≤ 72h** — the order is within 72 hours before an
  upcoming operative event, consistent with surgical preparation.
- **Delta-Hb trigger** (the acute-blood-loss override) — a documented
  Hb drop meets one of:
  - ≥ 1.5 g/dL drop within 6 hours
  - ≥ 2.0 g/dL drop within 12 hours
  - ≥ 2.5 g/dL drop within 24 hours

## Review flags before auto-appropriate

Two checks can stop a would-be sub-threshold `APPROPRIATE` row and send it
to review instead:

- **Hemodilution flagged** — Hb is below threshold, but ≥ 2 L crystalloid
  in the prior 4 hours makes the Hb value suspect.
- **Single-low-Hb-no-trend** — an isolated Hb < 8 g/dL has no prior 24-hour
  Hb trend to confirm the anemia pattern.

Vital signs (low BP, high HR) are **supporting evidence only**. They
never trigger a bypass on their own and never override the Hb tier.

## Threshold summary

| Cohort | Hb threshold for `APPROPRIATE` | Notes |
| --- | --- | --- |
| Default | < 7.0 g/dL | Fall-through threshold. |
| Cardiac surgery | < 7.5 g/dL | Recent cardiac surgery cohort. |
| Orthopedic + cardiac history | < 8.0 g/dL | Orthopedic procedure plus cardiac-history evidence. |
| ESRD + dialysis-med signal | < 8.0 g/dL | ESRD diagnosis plus active dialysis-context medication. |
| MTP | None | Bypasses directly to `APPROPRIATE`. |
| Active heme malignancy | None | Routes to `NEEDS_REVIEW`; LLM interprets context. |
| Unknown | None | Routes to `NEEDS_REVIEW`; the classifier refuses to assume a threshold. |

## Why vitals are not standalone

Hemoglobin is the published threshold in KCMH PR 17.2 and AABB 2023.
Vital-sign-driven transfusion is a discretionary clinical judgment
outside the audit's deterministic scope. Vitals enter the LLM evidence
bundle when a row routes to the LLM; they do not change the
deterministic classification.
