`bba ingest` is the entry point for every audit. It validates, hashes, and
materializes the input bundle into a DuckDB + Parquet snapshot under
`$BBA_DATA_DIR`.

## Expected tables

The HOSxP export bundle contains 11 canonical tables:

| Table | Used for |
| --- | --- |
| `BDVST` | Transfusion order header: HN, AN, REQNO, product, request datetime. |
| `BDVSTDT` | Order detail rows and product/unit-level context. |
| `BDTYPE` | Blood-product reference. |
| `BDVSTST` | Transfusion-status reference. |
| `Diagnosis` | ICD-10 diagnosis evidence for exclusions and cohorts. |
| `Lab` | Hb, ANC, and other lab evidence. |
| `Med` | Medication evidence: dialysis meds, chemo meds, crystalloid volume. |
| `IPDADMPROGRESS` | Progress-note evidence for the LLM bundle. |
| `IPDNRFOCUSDT` | Nursing-note evidence for the LLM bundle. |
| `IPTSUMOPRT` | Operative events and procedure timestamps. |
| `ICD9CM` | Procedure-code dictionary and OR/non-OR flag. |

The exact column contract is defined in `src/bba/ingest/`.

## What ingest writes

For each successful `bba ingest` run:

- Parquet snapshots under `$BBA_DATA_DIR/ingest/v1/<run_id>/parquet/`.
- A DuckDB database at `$BBA_DATA_DIR/ingest/v1/<run_id>/ingest.duckdb`.
- A completion marker; absence of this marker means the run is incomplete
  and a re-ingest will start fresh.

For `bba audit --input ...` in Phase 1, the same ingest snapshot shape is
written under `$BBA_DATA_DIR/audit/<run_id>/`.

## run_id derivation

```text
run_id = sha256(input_csv_hash + schema_fingerprint + code_version)
```

This is the key invariant: identical input + identical code = identical
`run_id`. The audit pipeline refuses to write a duplicate `(run_id, encounter_id)`
row to `audit_store` — see [Run-level idempotency](/blood_audit/en/operators/troubleshooting/#run-level-idempotency).

## Sample placeholder values

For examples in this site we use synthetic placeholders only:

```text
HN: PHI_xxx_001
AN: PHI_xxx_002
REQNO: PHI_xxx_003
```

Never substitute real HN / AN values in documentation, screenshots, or
issue reports.
