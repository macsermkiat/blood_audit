blood_audit enforces a strict 3-variable contract for the full
deployment. Missing or malformed values fail fast with `CliError` — they
never silently degrade.

## Which variables are needed today

| Variable | Required for Phase 1 `bba audit`? | First used by |
| --- | --- | --- |
| `BBA_DATA_DIR` | Yes — validated at every CLI invocation | `bba.cli.main._resolve_data_dir` |
| `BBA_DB_URL` | No — needed in Phase 1.5 | `bba.audit_pipeline.store.PostgresBatchRunStore`, `bba.review_actions` |
| `ANTHROPIC_API_KEY` | No — needed in Phase 1.5 | `bba.llm_client` (Anthropic Batch API) |

Set all three when configuring a long-lived deployment. Only
`BBA_DATA_DIR` is strictly required for running the Phase 1 ingest leg
locally.

## Important details

### `BBA_DATA_DIR`

Must be an **absolute path**. The directory must exist and be writable by
the service user. Validated by `_resolve_data_dir()` in
`src/bba/cli/main.py`. Holds:

- DuckDB ingest snapshots
- Parquet audit rows (cold storage)
- Run-state completion markers used by idempotency

### `BBA_DB_URL`

Postgres DSN. Always use the `+psycopg` driver suffix — SQLAlchemy
routes a bare `postgresql://` URL to psycopg2 by default, which is **not**
the driver this project ships. Canonical converter:
`bba.review_actions.models.ReviewActionsConfig.sqlalchemy_dsn`.

Example:

```text
postgresql+psycopg://bba_user@db.internal:5432/blood_audit
```

### `ANTHROPIC_API_KEY`

Required before `bba audit` invokes the Batch API. The cost guard in
`src/bba/audit_pipeline/cost_guard.py` rejects live API access in tests
to prevent accidental Batch submission during development.

## Verify

```bash
env | grep -E '^(BBA_DATA_DIR|BBA_DB_URL|ANTHROPIC_API_KEY)='
```

All three lines should print. Values for `ANTHROPIC_API_KEY` should never
be logged; redact before sharing terminal output.
