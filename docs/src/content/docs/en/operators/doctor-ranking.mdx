## What it is

`bba.attribution` (Feature 2, PR #82) ranks the top-10 ordering doctors and
departments by blood-order appropriateness, in three buckets: Appropriate,
Inappropriate, and Unresolved.

The attribution chain resolves an order back to a doctor:

- An order's `REQNO` → `BDVST.DCTREQ` → the `DCT.csv` doctor registry.

The verdict source is **swappable** through the `VerdictSource` seam and
selectable at runtime with `BBA_VERDICT_SOURCE`: the default `human`
ranks on the 300-case human-review workbook (Sheet1, column J), and
`pipeline` ranks on full-cohort pipeline verdicts read from the audit
store (`BBA_AUDIT_STORE_DIR` + `BBA_RUN_ID` / `BBA_CODE_VERSION`).
With the pipeline source, scorecards also carry the returns terminals
(`RETURNED_NOT_TRANSFUSED` / `PERIOP_TRANSFUSION_EXEMPT`) as an
*excluded* bucket and fold `PREOP_OVER_RESERVATION` into the
inappropriate side while counting it separately as
`over_reservation_count`.

This is a **pilot script / integration seam**, not a top-level `bba`
subcommand. It is also surfaced through the `serve-dashboard` integration seam
(the attribution resolvers).

## How to run

The pilot script is `scripts/pilot/rank_doctors.py`. From the repo root:

```bash
uv run python scripts/pilot/rank_doctors.py
```

The script **fails loud** (exits non-zero) if any of the three input files is
missing, printing which one.

### Environment variables

| Variable | What it is | Default |
| --- | --- | --- |
| `BBA_REVIEW_XLSX` | The human-review workbook (verdict source). | `~/Downloads/Review การใช้เลือด.xlsx` |
| `BBA_BDVST_CSV` | BDVST export containing `REQNO` + `DCTREQ`. | `../Bloodbank/data/encrypted/BDVST.csv` (relative to repo root) |
| `BBA_DCT_CSV` | The `DCT.csv` doctor registry. | `../Bloodbank/data/raw/DCT.csv` |
| `BBA_PILOT_WORK_DIR` | Output directory. | `/tmp/bba_mini` |
| `BBA_VERDICT_SOURCE` | Verdict source: `human` or `pipeline`. | `human` |
| `BBA_AUDIT_STORE_DIR` | Audit-store directory (pipeline source only). | — |
| `BBA_RUN_ID` | run_id scope for store reads (pipeline source only). | — |
| `BBA_CODE_VERSION` | code-version scope for store reads (pipeline source only). | — |
| `BBA_PILOT_REPORT_CSV` | The pilot `report.csv` input (pipeline source only). | `$BBA_PILOT_WORK_DIR/report.csv` |

### Example invocation

```bash
export BBA_REVIEW_XLSX="$HOME/Downloads/Review การใช้เลือด.xlsx"
export BBA_BDVST_CSV=/path/to/BDVST.csv
export BBA_DCT_CSV=/path/to/DCT.csv
export BBA_PILOT_WORK_DIR=/tmp/bba_mini
uv run python scripts/pilot/rank_doctors.py
open "$BBA_PILOT_WORK_DIR/doctor_rankings.html"
```

## Outputs

Written to `$BBA_PILOT_WORK_DIR`:

| Output | What it is |
| --- | --- |
| `doctor_ranking.csv` | Top-10 doctors, ranked with per-bucket counts. |
| `department_ranking.csv` | Top-10 departments, ranked with per-bucket counts. |
| `doctor_rankings.html` | Self-contained page for human review. |

The script also prints a **reconciliation summary** to stdout: total verdicts
(with appropriate / inappropriate / unresolved counts), the ranked bucket, the
minimum-N threshold, and the number of doctor/department table rows.

## Where this lives in code

- `src/bba/attribution/` — the tested module (loaders, aggregation, ranking,
  writers).
- `scripts/pilot/rank_doctors.py` — thin glue that resolves the input paths and
  reports the reconciliation summary.
