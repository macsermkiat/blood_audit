## คำสำคัญ

- `AuditRow` — แถวผลลัพธ์หนึ่งรายการต่อคำสั่งให้เลือดที่ audit
- `final_classification` — classification ที่เก็บและใช้ในรายงาน
- `rule_classification` — ผลจาก deterministic classifier ก่อน LLM/finalization
- `LLM_REVIEW` — label สำหรับ routing ภายใน ไม่ใช่ classification ที่เก็บ
- `hallucination_suspect` — `review_reason` สำหรับ quote-grounding failure
- `run_id` — hash จาก input + schema + code version เพื่อ idempotency
- `SafeId` — placeholder ที่ใช้แทน PHI เช่น `PHI_xxx_001`

## Module สำคัญ

| Module | หน้าที่ |
| --- | --- |
| `bba.ingest` | อ่าน HOSxP CSV, ตรวจ schema, normalize rows และเขียน completion marker ใน Phase 1 |
| `bba.hb_lookup` | หา Hb ก่อนให้เลือดและ prior Hb trend |
| `bba.cohort_detector` | กำหนด cohort หรือบริบททางคลินิกของคำสั่ง |
| `bba.deterministic_classifier` | กฎ 3-tier Hb, bypass และ review flags |
| `bba.deid_redactor` | de-identify evidence bundle |
| `bba.quote_grounder` | ตรวจ quote ของ LLM แบบ fail-closed |
| `bba.audit_store` | persistence, idempotency และ audit trail |
