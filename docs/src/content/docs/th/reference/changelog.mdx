## 0.1.0

- Ingest HOSxP CSV: ตรวจ schema, normalize rows และบันทึก run markers
- Deterministic 3-tier Hb classifier พร้อม cohort-aware thresholds
- Evidence bundle, de-id, prompt builder และ LLM batch modules
- Quote-grounding 6 ชั้นแบบ fail-closed
- Audit store models, idempotency และ review-action primitives
- Docs site สำหรับ operator, developer และ clinical/QI users

## สิ่งที่เพิ่มหลังจาก Phase 1 รอบแรก

- **Phase 2 — platelet auditor (PR #85, merged)** `component_map` แยกประเภท
  ส่วนประกอบเลือด (RED_CELL / PLATELET / FFP / CRYO / WHOLE_BLOOD / UNKNOWN);
  `platelet_lookup` + `platelet_classifier` เพิ่ม §5.1 count gate ที่
  **ไม่ auto-clear อะไรเลย**: count ≥ 100 ×10³/µL → review, ต่ำกว่านั้น →
  ส่งต่อ LLM, ไม่มีค่า → INSUFFICIENT_EVIDENCE. Platelet LLM leg อยู่หลัง flag
  `PLATELET_LLM_ENABLED` ที่ปิดโดยค่าเริ่มต้น; RBC path เหมือนเดิมทุกไบต์
- **Doctor / department attribution (Feature 2, PR #82)** `bba.attribution`
  จัดอันดับแพทย์และแผนกที่สั่งเลือด 10 อันดับแรกตามความเหมาะสม แบ่งเป็น 3 กลุ่ม
  ผ่าน `BDVST.DCTREQ` → `DCT.csv` โดยแหล่ง verdict สลับเปลี่ยนได้
- **ปรับ cohort** แยก `ORTHO_SURGERY` เป็น cohort ของตัวเอง (Hb floor 8.0,
  PR #83); เพิ่ม `CARDIOPULMONARY_COMORBIDITY` cohort + peri-op defer-to-LLM
  (PR #81)
- **Stale-volume gate ของ bleeding auto-clear** ข้อยกเว้นแบบ prose เดียวของ
  RBC over-clear guardrail (quote `ACTIVE_BLEEDING` ที่ ground แล้ว confidence
  ≥ 0.8 พร้อมปริมาณ > 300 mL หรือ marker ที่เป็นอันตรายถึงชีวิต) จะลบช่วง
  ข้อความที่อยู่ใต้ date anchor แบบ `d/m/y` (Thai BE หรือ CE) ที่เก่ากว่า 7 วัน
  ก่อนการสั่งเลือด ทำให้ปริมาณเลือดออกของ index bleed เมื่อหลายสัปดาห์ก่อน
  ไม่สามารถ clear order ปัจจุบันได้อีกต่อไป ข้อความที่ไม่มีวันที่หรืออ่านไม่ได้
  จะไม่ถูกแตะต้อง — gate นี้ได้เพียงระงับข้อยกเว้นเท่านั้น RBC prompt ระบุกฎ
  current-episode เดียวกันเพื่อให้ verdict ของโมเดลตรงกับ guardrail
