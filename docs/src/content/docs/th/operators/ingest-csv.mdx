`bba ingest` คือจุดเริ่มต้นสำหรับตรวจชุดข้อมูลส่งออกจาก HOSxP ใน Phase 1
คำสั่งนี้ตรวจ header ตาม schema, คำนวณ hash, stream แถวผ่าน normalization
และเขียน completion marker ภายใต้ `$BBA_DATA_DIR` ส่วน DuckDB + Parquet
เป็นรูปแบบ storage ที่ตั้งใจจะเชื่อมต่อถัดไป แต่ `bba ingest` ยังไม่เขียน

## ตารางที่ต้องมี

ชุดข้อมูลส่งออกจาก HOSxP มีตารางหลัก 12 ตาราง:

| ตาราง | ใช้ทำอะไร |
| --- | --- |
| `BDVST` | header ของคำสั่งเลือด: HN, AN, REQNO, ชนิดผลิตภัณฑ์, วันเวลา request, แพทย์ผู้สั่ง (DCTREQ) |
| `BDVSTDT` | รายละเอียดคำสั่งและบริบทระดับ unit/product |
| `BDTYPE` | reference ของชนิดผลิตภัณฑ์เลือด |
| `BDVSTST` | reference ของสถานะการให้เลือด |
| `Diagnosis` | หลักฐาน ICD-10 สำหรับ exclusion และบริบททางคลินิก |
| `Lab` | Hb, ANC และ lab อื่นที่เกี่ยวข้อง |
| `Med` | หลักฐานยา: dialysis med, chemo med, ปริมาณ crystalloid |
| `IPDADMPROGRESS` | progress note สำหรับ evidence bundle ของ LLM |
| `IPDNRFOCUSDT` | nursing note สำหรับ evidence bundle ของ LLM |
| `IPTSUMOPRT` | operative event และเวลา procedure |
| `INCPT_OPRTACT` | ตาราง procedure charge-support: export ที่ pre-join INCPT ⋈ OPRTACT บน Income (ตาม #69) และถูก validate เป็นส่วนหนึ่งของ schema v1 |
| `ICD9CM` | dictionary ของ procedure code และ flag ว่าเป็น OR/non-OR |

สัญญาของคอลัมน์จริงอยู่ใน `src/bba/ingest/`

## สิ่งที่ ingest เขียนออก

เมื่อ `bba ingest` สำเร็จใน Phase 1 จะได้:

- `$BBA_DATA_DIR/ingest/v1/` เป็น root ของ ingest output
- `_run_<run_id>.complete` marker เพื่อบอกว่า schema validation และ
  row-normalization drain สำเร็จแล้ว
- ถ้า marker นี้หายไป การ ingest ซ้ำจะเริ่มใหม่อย่างปลอดภัย

สำหรับ `bba audit --input ...` ใน Phase 1 จะรัน ingest validation/drain
ภายใต้ `$BBA_DATA_DIR/audit/<run_id>/` และบันทึก run-state หนึ่งรายการต่อ
ตาราง HOSxP ที่ validate ผ่าน

## การสร้าง run_id

```text
run_id = sha256(input_csv_hash + schema_fingerprint + code_version)
```

หลักสำคัญคือ input เดิม + schema เดิม + code เดิม ต้องได้ `run_id` เดิม
เสมอ เพื่อให้การรันซ้ำไม่เขียนผลซ้ำโดยไม่ตั้งใจ

## ตัวอย่าง placeholder

ตัวอย่างในเอกสารนี้ใช้ placeholder เท่านั้น:

```text
HN: PHI_xxx_001
AN: PHI_xxx_002
REQNO: PHI_xxx_003
```

ห้ามแทนค่าด้วย HN, AN หรือ REQNO จริงในเอกสาร screenshot หรือ issue report
