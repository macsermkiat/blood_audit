Integration seam คือจุดที่โค้ดภายในพร้อมแล้ว แต่การเรียกจาก CLI ยังต้องรอ
ส่วนเชื่อมกับระบบจริงของ deployment เช่น database, scheduler, policy store
หรือ key management จึงตั้งใจให้ fail loud แทนการทำงานด้วยค่า default ปลอม

(`bba report` เคยเป็น seam ตัวที่สี่ ตอนนี้ **เชื่อมแล้ว** — ใช้ render
CSV สำหรับคณะกรรมการ, CSV รายแพทย์ และ PDF จาก audit store โดย resolve
attribution ของ ward/แพทย์จาก `BBA_BDVST_CSV` / `BBA_DCT_CSV`)

## สิ่งที่ไม่ควรแก้

- อย่า swallow `CliError` เพื่อให้คำสั่งดูเหมือนสำเร็จ
- อย่าเพิ่ม mock database หรือ fake API key ใน production path
- อย่าทำ dashboard ที่อ่าน PHI โดยไม่ผ่าน auth และ de-identification boundary

หากต้องการทดสอบ ให้เขียน test fixture แยกจาก production CLI path แทน
