## คืออะไร

`bba.attribution` (Feature 2, PR #82) จัดอันดับแพทย์ผู้สั่งและแผนก 10 อันดับแรก
ตามความเหมาะสมของการสั่งใช้เลือด แบ่งเป็นสามกลุ่ม: เหมาะสม, ไม่เหมาะสม และยังไม่สรุป

สาย attribution จะย้อนคำสั่งกลับไปหาแพทย์:

- `REQNO` ของคำสั่ง → `BDVST.DCTREQ` → ทะเบียนแพทย์ `DCT.csv`

แหล่งที่มาของ verdict **สลับได้** ผ่าน seam ชื่อ `VerdictSource` และเลือกได้
ตอนรันด้วย `BBA_VERDICT_SOURCE`: ค่าเริ่มต้น `human` จัดอันดับจาก workbook
การตรวจโดยคน 300 เคส (Sheet1, คอลัมน์ J) ส่วน `pipeline` จัดอันดับจาก verdict
ของ pipeline เต็มทั้ง cohort ที่อ่านจาก audit store (`BBA_AUDIT_STORE_DIR` +
`BBA_RUN_ID` / `BBA_CODE_VERSION`) เมื่อใช้แหล่ง pipeline, scorecard จะมี
terminal ฝั่งคืนเลือด (`RETURNED_NOT_TRANSFUSED` / `PERIOP_TRANSFUSION_EXEMPT`)
เป็นกลุ่ม *ไม่นับคะแนน (excluded)* ด้วย และรวม `PREOP_OVER_RESERVATION`
เข้าฝั่งไม่เหมาะสม โดยนับแยกไว้เป็น `over_reservation_count`

ตัวนี้เป็น **สคริปต์ pilot / integration seam** ไม่ใช่ subcommand ระดับบนของ
`bba` และยังถูกนำเสนอผ่าน integration seam ของ `serve-dashboard`
(attribution resolvers) อีกด้วย

## วิธีรัน

สคริปต์ pilot คือ `scripts/pilot/rank_doctors.py` จาก root ของ repo:

```bash
uv run python scripts/pilot/rank_doctors.py
```

สคริปต์ **fail loud** (exit ไม่เป็นศูนย์) ถ้าไฟล์ input ตัวใดตัวหนึ่งในสามหายไป
พร้อมพิมพ์ว่าไฟล์ไหนที่หาย

### ตัวแปรสภาพแวดล้อม

| ตัวแปร | คืออะไร | ค่าเริ่มต้น |
| --- | --- | --- |
| `BBA_REVIEW_XLSX` | workbook การตรวจโดยคน (แหล่งของ verdict) | `~/Downloads/Review การใช้เลือด.xlsx` |
| `BBA_BDVST_CSV` | ไฟล์ส่งออก BDVST ที่มี `REQNO` + `DCTREQ` | `../Bloodbank/data/encrypted/BDVST.csv` (สัมพัทธ์กับ root ของ repo) |
| `BBA_DCT_CSV` | ทะเบียนแพทย์ `DCT.csv` | `../Bloodbank/data/raw/DCT.csv` |
| `BBA_PILOT_WORK_DIR` | โฟลเดอร์ผลลัพธ์ | `/tmp/bba_mini` |
| `BBA_VERDICT_SOURCE` | แหล่งที่มาของ verdict: `human` หรือ `pipeline` | `human` |
| `BBA_AUDIT_STORE_DIR` | ไดเรกทอรีของ audit store (เฉพาะแหล่ง pipeline) | — |
| `BBA_RUN_ID` | ขอบเขต run_id สำหรับอ่านจาก store (เฉพาะแหล่ง pipeline) | — |
| `BBA_CODE_VERSION` | ขอบเขต code version สำหรับอ่านจาก store (เฉพาะแหล่ง pipeline) | — |
| `BBA_PILOT_REPORT_CSV` | ไฟล์ `report.csv` ราย order ของ pilot (ให้ค่า mean pre-transfusion trigger) จำเป็นสำหรับ**ทุก** run ไม่ว่า human หรือ pipeline — ถ้าไฟล์หายจะ fail loud | `$BBA_PILOT_WORK_DIR/report.csv` |

### ตัวอย่างการรัน

```bash
export BBA_REVIEW_XLSX="$HOME/Downloads/Review การใช้เลือด.xlsx"
export BBA_BDVST_CSV=/path/to/BDVST.csv
export BBA_DCT_CSV=/path/to/DCT.csv
export BBA_PILOT_WORK_DIR=/tmp/bba_mini
uv run python scripts/pilot/rank_doctors.py
open "$BBA_PILOT_WORK_DIR/doctor_rankings.html"
```

## ผลลัพธ์

เขียนไปที่ `$BBA_PILOT_WORK_DIR`:

| ผลลัพธ์ | คืออะไร |
| --- | --- |
| `doctor_ranking.csv` | แพทย์ 10 อันดับแรก จัดอันดับพร้อมจำนวนต่อกลุ่ม |
| `department_ranking.csv` | แผนก 10 อันดับแรก จัดอันดับพร้อมจำนวนต่อกลุ่ม |
| `doctor_rankings.html` | หน้าเดียวจบสำหรับให้คนตรวจ |

สคริปต์ยังพิมพ์ **สรุปการกระทบยอด (reconciliation summary)** ไปที่ stdout:
จำนวน verdict ทั้งหมด (พร้อมจำนวนเหมาะสม / ไม่เหมาะสม / ยังไม่สรุป), กลุ่มที่จัดอันดับ,
ค่า threshold ขั้นต่ำ (minimum-N) และจำนวนแถวตารางของแพทย์/แผนก

## อยู่ตรงไหนในโค้ด

- `src/bba/attribution/` — module ที่มีเทสต์ (loaders, aggregation, ranking,
  writers)
- `scripts/pilot/rank_doctors.py` — glue บาง ๆ ที่ resolve path ของ input และ
  รายงานสรุปการกระทบยอด
