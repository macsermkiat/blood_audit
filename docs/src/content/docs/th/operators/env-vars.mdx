## รายละเอียดที่สำคัญ

### `BBA_DATA_DIR`

ต้องเป็น **absolute path** ไดเรกทอรีต้องมีอยู่จริง และ service user
ต้องเขียนได้ ฟังก์ชัน `_resolve_data_dir()` ใน `src/bba/cli/main.py`
จะตรวจเงื่อนไขนี้ ใช้เก็บข้อมูล:

- run markers ของ ingest และ audit ใน Phase 1
- Parquet audit rows ใน Phase 1.5 เมื่อเปิดใช้ storage wiring
- completion markers สำหรับ idempotency

### `BBA_DB_URL`

Postgres DSN ต้องใช้ driver suffix `+psycopg` เสมอ เพราะ SQLAlchemy
จะ route `postgresql://` แบบไม่มี suffix ไปยัง psycopg2 ซึ่ง **ไม่ใช่** driver
ที่โปรเจกต์นี้ใช้ มีตัวแปลงมาตรฐานที่
`bba.review_actions.models.ReviewActionsConfig.sqlalchemy_dsn`

ตัวอย่าง:

```text
postgresql+psycopg://bba_user@db.internal:5432/blood_audit
```

### `ANTHROPIC_API_KEY`

จำเป็นก่อนที่ `bba audit` จะเรียก Batch API ได้ ส่วน cost guard ที่
`src/bba/audit_pipeline/cost_guard.py` จะปฏิเสธ live API access ระหว่างเทสต์
เพื่อป้องกันการส่ง batch โดยไม่ตั้งใจระหว่างพัฒนา

### `BBA_BDVST_CSV` / `BBA_DCT_CSV` (สำหรับ `bba report`)

คำสั่ง `bba report` ที่เชื่อมแล้วต้องใช้สองตัวแปรนี้สำหรับ
ward/physician attribution resolvers (`_get_ward_resolver` /
`_get_physician_resolver` ใน `src/bba/cli/main.py`) — ถ้าไม่ตั้งจะ
fail loud ด้วย `CliError`

## ตัวแปรเฉพาะ pilot

ส่วน pilot (เส้นทาง A / เส้นทาง C) อ่านตัวแปร `BBA_PILOT_*` ชุดที่กว้างกว่า
รวมถึง `BBA_BDVSTTRANS_CSV` (override ไฟล์ส่งออก returns ledger) และ
kill-switch ของ flag สำหรับฟีเจอร์ที่ default-ON
(`BBA_PILOT_MSBOS_RESERVATION=0`, `BBA_PILOT_DECLARED_USETYPE=0`,
`BBA_PILOT_DECLARED_USE_PREOP_EXEMPT=0`,
`BBA_PILOT_MSBOS_PLANNED_OP_PICKER_V2=0`) ตัวแปรเหล่านี้มีเอกสารอยู่ที่
[รัน pipeline](/blood_audit/th/operators/run-pipeline/) และ
[จัดอันดับแพทย์และแผนก](/blood_audit/th/operators/doctor-ranking/)
ส่วนตารางด้านบนครอบคลุมเฉพาะ CLI ที่รองรับเท่านั้น

## ตรวจสอบ

```bash
env | grep -E '^(BBA_DATA_DIR|BBA_DB_URL|ANTHROPIC_API_KEY|BBA_BDVST_CSV|BBA_DCT_CSV)='
```

ควรเห็นค่าที่จำเป็นครบตาม phase ที่ใช้งาน (ครบทั้งห้าตัวสำหรับ
deployment เต็มที่รัน `bba report`) ห้าม log ค่า
`ANTHROPIC_API_KEY`; ลบออกก่อนแชร์ output จาก terminal เสมอ
