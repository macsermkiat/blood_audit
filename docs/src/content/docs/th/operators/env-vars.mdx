## รายละเอียดที่สำคัญ

### `BBA_DATA_DIR`

ต้องเป็น **absolute path** ไดเรกทอรีต้องมีอยู่จริง และ service user
ต้องเขียนได้ ฟังก์ชัน `_resolve_data_dir()` ใน `src/bba/cli/main.py`
จะตรวจเงื่อนไขนี้ ใช้เก็บข้อมูล:

- DuckDB ingest snapshots
- Parquet audit rows (cold storage)
- Run-state completion markers สำหรับ idempotency

### `BBA_DB_URL`

Postgres DSN ต้องใช้ driver suffix `+psycopg` เสมอ เพราะ SQLAlchemy
จะ route `postgresql://` แบบไม่มี suffix ไปยัง psycopg2 ซึ่ง **ไม่ใช่** driver
ที่โปรเจกต์นี้ใช้ มีตัวแปลงมาตรฐานที่
`bba.review_actions.models.ReviewActionsConfig.sqlalchemy_dsn`

ตัวอย่าง:

```text
postgresql+psycopg://bba_user@db.internal:5432/blood_audit
```

### `ANTHROPIC_API_KEY`

จำเป็นก่อนที่ `bba audit` จะเรียก Batch API ได้ ส่วน cost guard ที่
`src/bba/audit_pipeline/cost_guard.py` จะปฏิเสธ live API access ระหว่างเทสต์
เพื่อป้องกันการส่ง batch โดยไม่ตั้งใจระหว่างพัฒนา

## ตรวจสอบ

```bash
env | grep -E '^(BBA_DATA_DIR|BBA_DB_URL|ANTHROPIC_API_KEY)='
```

ควรเห็นค่าที่จำเป็นครบตาม phase ที่ใช้งาน ห้าม log ค่า
`ANTHROPIC_API_KEY`; ลบออกก่อนแชร์ output จาก terminal เสมอ
