## ใช้ `--input` หรือ `--run-id` เมื่อไร

- `--input INPUT_CSV` — ใช้เมื่อรันครั้งแรกกับ bundle นี้ ระบบจะคำนวณ
  `run_id` จาก input hash + schema fingerprint + code version แล้ว ingest
  และ mark run complete
- `--run-id RUN_ID` — ใช้กลับไปอ้างถึง run ที่ complete แล้ว หากไม่ใส่
  `--force` จะเป็น no-op เพราะ run นั้นสมบูรณ์อยู่แล้ว

## `--force` ต้องมี audit trail

เมื่อใช้ `--force` คำสั่งจะเขียน `audit_log` พร้อม
`idempotency_override=true` เพื่อบอกว่ามีการบังคับรันซ้ำ ห้ามแก้ไฟล์
run-state โดยตรงเพื่อเลี่ยงกลไกนี้ เพราะจะทำให้ audit trail ไม่ครบ

## Idempotency ระดับ run

ถ้ามี CLI สอง process เรียก run เดียวกันพร้อมกัน ระบบจะ serialize ผ่าน
`store.acquire_run_lock` จึงไม่สามารถ ingest run เดียวกันซ้ำพร้อมกันได้
แต่ audit คนละ input สามารถรันคู่ขนานได้

## Phase 1.5 แบบย่อ

เมื่อเชื่อม LLM analysis แล้ว คำสั่งนี้จะ:

- สร้าง evidence bundle, de-identify และส่งแถวที่ต้อง review ไปยัง
  Anthropic Batch API
- ตรวจคำตอบ LLM ทุกแถวด้วย quote-grounding แบบ fail-closed 6 ชั้น
- เขียน `AuditRow` 1 แถวต่อคำสั่งให้เลือด พร้อม `rule_classification`
  และ `final_classification`

สถานะ batch จะเก็บใน `batch_runs` ต่อคำสั่งให้เลือด
(`pending` → `submitted` → `partial` → `complete`; `failed` เป็น terminal)
ถ้าระบบล่มกลางทาง การรันครั้งถัดไปจะ resume ได้

## ขอบเขตความปลอดภัย

- อ่านเฉพาะ HOSxP export และเขียนผลภายใต้ `$BBA_DATA_DIR/audit/<run_id>/`
  ไม่แก้ตาราง HOSxP ต้นทาง
- ทำงานย้อนหลังและเงียบเสมอ ไม่มี real-time alert ไปยังแพทย์
- `run_id` ที่ complete แล้วจะไม่ถูกเขียนทับแบบเงียบ ๆ
