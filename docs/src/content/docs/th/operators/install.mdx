## สิ่งที่ต้องมี

| สิ่งที่ต้องมี | เวอร์ชัน |
| --- | --- |
| Python | ≥ 3.13 |
| uv | ≥ 0.9.24 |
| PostgreSQL | ≥ 14 (driver psycopg3) |
| Docker (สำหรับเทสต์เท่านั้น) | latest |

## Clone และ sync dependencies

```bash
git clone https://github.com/macsermkiat/blood_audit.git
cd blood_audit
uv sync --frozen
```

Lockfile ถูก freeze ไว้แล้ว **ห้ามตั้ง `UV_LOCKED=1`** เพราะจะขัดแย้งกับ
`--frozen` ใน uv ≥ 0.9

## ตรวจสอบ CLI

```bash
uv run bba --help
```

จะเห็น subcommand 6 ตัว โดย 4 ตัว คือ `evaluate`, `report`,
`serve-dashboard` และ `sentinel` จะ raise `CliError` โดยตั้งใจ
**ไม่ใช่บั๊ก** ดูรายละเอียดที่ [Integration seams](/blood_audit/th/operators/integration-seams/)

## ตั้งค่า GitHub Pages ครั้งเดียว (เฉพาะการเผยแพร่เอกสารนี้)

ข้ามส่วนนี้ได้ หากคุณกำลังดูแล pipeline ไม่ได้ดูแลการเผยแพร่เว็บเอกสาร

ถ้าจะเผยแพร่เว็บเอกสารจาก fork ของคุณเอง:

1. Settings ของ repo → **Pages** → Source: **GitHub Actions**
2. Merge การเปลี่ยนแปลงที่แตะ `docs/**` เข้า `main`
3. Workflow `deploy-docs` จะ build และ deploy ให้อัตโนมัติ
   URL ของ Pages จะปรากฏใน run summary
