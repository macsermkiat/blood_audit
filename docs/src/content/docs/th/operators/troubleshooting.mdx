## `BBA_DATA_DIR` ใช้ไม่ได้

ตรวจว่าเป็น absolute path, directory มีอยู่จริง และ user ที่รัน service
เขียนได้:

```bash
echo "$BBA_DATA_DIR"
test -d "$BBA_DATA_DIR"
test -w "$BBA_DATA_DIR"
```

ถ้า path เป็น relative หรือเขียนไม่ได้ CLI จะหยุดตั้งแต่เริ่มด้วย
`CliError`

## Postgres DSN ผิดรูปแบบ

สำหรับ Phase 1.5 ต้องใช้:

```text
postgresql+psycopg://user@host:5432/dbname
```

อย่าใช้ `postgresql://` แบบเปล่า เพราะ SQLAlchemy จะพาไปใช้ psycopg2
ซึ่งไม่ใช่ driver ที่โครงการนี้เลือกใช้

## Subcommand บางตัว fail ทันที

`evaluate`, `report`, `serve-dashboard` และ `sentinel` fail โดยตั้งใจใน
Phase 1 เพราะยังเป็น integration seams ดูรายละเอียดที่
[Integration seams](/blood_audit/th/operators/integration-seams/)

## Run-level idempotency

ถ้ารัน input เดิมและ code เดิม จะได้ `run_id` เดิมและไม่เขียน run marker หรือผลลัพธ์ซ้ำ
นี่เป็นพฤติกรรมที่ถูกต้อง หากต้องการบังคับรันซ้ำจริง ๆ ให้ใช้ `--force`
เพื่อให้ระบบบันทึก audit trail ว่ามีการ override idempotency
