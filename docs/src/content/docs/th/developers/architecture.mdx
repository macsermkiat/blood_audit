## แนวคิดหลัก

ระบบแยกเป็น stage ชัดเจน แต่ละ stage รับข้อมูล typed object หรือข้อมูลที่ตรวจสอบแล้ว
จาก stage ก่อนหน้า แล้วส่งต่อผลลัพธ์ที่ตรวจสอบได้ จุดที่รวมหลาย stage เข้าด้วยกัน
ควรอยู่ใน `bba.audit_pipeline` เท่านั้น เพื่อให้ boundary ของโมดูลอื่นยังเล็ก
และทดสอบได้ง่าย

Intake แยกที่ `bba.component_map`: คำสั่งเลือดแดงเข้ากับ RBC path 12 stage
(Phase 1) ส่วนคำสั่งเกล็ดเลือดเข้ากับ platelet leg (Phase 2 — merged, PR #85)
โดย RBC path เหมือนเดิมทุกไบต์ไม่ว่าจะเปิดฟีเจอร์เกล็ดเลือดหรือไม่

## กลุ่ม package สำคัญ

รายชื่อเต็ม 30 modules อยู่ใน [Module glossary](/blood_audit/th/reference/module-glossary/)
(อ้างอิงจาก `docs/CONTEXT.md`) ตารางนี้แสดงเฉพาะ package หลัก:

| Package | หน้าที่ |
| --- | --- |
| `bba.ingest` | ตรวจ CSV schema, hash input, normalize rows และเขียน completion marker |
| `bba.audit_orders` | แปลงคำสั่งเลือดให้เป็น audit anchor ต่อ order; รับคำสั่งเกล็ดเลือดแยกจากสถิติ RBC |
| `bba.component_map` | intake gate: แยก component family (RED_CELL, PLATELET, FFP, CRYO, WHOLE_BLOOD, UNKNOWN) และ fork RBC/platelet legs |
| `bba.hb_lookup` | หา Hb ก่อนให้เลือดและ prior Hb trend |
| `bba.cohort_detector` | assign clinical-context cohort เช่น default, cardiac surgery, ortho-surgery, ESRD-EPO, cardiopulmonary-comorbidity (heart disease), MTP, unknown |
| `bba.deterministic_classifier` | ใช้กฎ 3-tier Hb และ bypass/review rules; ปล่อย routing terminals `RETURNED_NOT_TRANSFUSED` / `PERIOP_TRANSFUSION_EXEMPT` ได้ด้วย และไม่ปล่อย `INAPPROPRIATE` เอง (LLM-only) |
| `bba.evidence_bundle_builder` | รวมหลักฐานสำหรับแถวที่ต้องเข้า LLM route |
| `bba.deid_redactor` | de-identify bundle ก่อนออกจาก container |
| `bba.quote_grounder` | ตรวจ citation ของ LLM แบบ fail-closed |
| `bba.audit_store` | เขียน AuditRow, LlmCall และ batch/run state |
| `bba.platelet_classifier` | §5.1 platelet gate; ไม่ auto-clear อะไรเลย (≥ 100 → review, &lt; 100 → defer-LLM, ไม่มีค่า → `INSUFFICIENT_EVIDENCE`) |
| `bba.attribution` | จัดอันดับแพทย์/แผนก 10 อันดับตามความเหมาะสม (`BDVST.DCTREQ` → `DCT.csv`) |
| `bba.returns_ledger` | รวม disposition ต่อ unit จาก BDVSTTRANS → `not_transfused` / `transfused` / `inconclusive`; ป้อน returns terminals เปิดเป็น default-ON |
| `bba.declared_use` | ตีความ `BDVSTDT.USETYPE` ตอนสั่ง (declared surgical / type-and-screen intent); เป็น pre-op router เดียวใน default mode เปิดเป็น default-ON |
| `bba.preop_reservation` | ประเมิน MSBOS reservation: reference loader, planned-op picker v2, OPRTACT→ICD9 bridge, RBC + platelet legs และ reason vocabulary (`reason_presentation`) เปิดเป็น default-ON |
| `bba.feature_flags` | โมดูล runtime flag (7 flags; pilot legs ใช้ env override `BBA_PILOT_*`) |

## เส้นทาง failure

Validation, de-identification, quote grounding และ persistence ต้อง fail loud
ห้าม silently pass เพราะ output ของระบบนี้ถูกใช้ในการออดิท QI หากไม่แน่ใจ
ให้ส่ง `NEEDS_REVIEW` แทนการ auto-classify
