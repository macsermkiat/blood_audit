## Classification ที่เก็บจริง

`final_classification` เก็บได้ 5 ค่า:

- `APPROPRIATE`
- `INAPPROPRIATE`
- `NEEDS_REVIEW`
- `INSUFFICIENT_EVIDENCE`
- `POTENTIALLY_INAPPROPRIATE`

`LLM_REVIEW` เป็น routing label ภายใน deterministic classifier ไม่ใช่ค่าที่เก็บ
ส่วน `hallucination_suspect` เป็น `review_reason` ไม่ใช่ classification

## Idempotency

การเขียนซ้ำด้วย `(audit_id, run_id, code_version)` เดิมต้องไม่สร้างแถวซ้ำ
เพราะ audit report ต้อง reproducible และตรวจย้อนหลังได้
